## Version CV longue

Assistant IA de Surveillance Continue — Projet personnel

Conception et développement d'un prototype complet de surveillance de machines industrielles simulées. Le projet génère des données capteurs réalistes, simule un flux temps réel, applique un pipeline de traitement continu avec Pathway, entraîne un modèle Isolation Forest pour détecter les anomalies, stocke les événements en SQLite, expose une API FastAPI et propose un dashboard Streamlit. Un agent IA rule-based explique les anomalies et formule des recommandations techniques, avec une architecture pensée pour évoluer vers LangGraph ou un LLM.

Version V2 : ajout d'un bootstrap de démonstration en une commande, d'un dashboard multi-pages orienté recruteur, d'une CI GitHub Actions, d'une documentation d'architecture et d'un scénario de démonstration en entretien.

## Pitch entretien 30 secondes

J'ai développé un assistant IA de monitoring industriel sur données simulées pour montrer une architecture Data/IA de bout en bout. Le projet couvre la génération de données capteurs, le streaming avec Pathway, la détection d'anomalies avec Isolation Forest, le stockage SQLite, une API FastAPI, un dashboard Streamlit et un agent explicatif. L'objectif était de construire une V1 réaliste, lançable et documentée, sans prétendre utiliser de vraies données industrielles.

En V2, j'ai renforcé la partie portfolio avec une démo en une commande, un dashboard plus professionnel, des tests automatisés GitHub Actions et une documentation pensée pour un recruteur technique.

## Compétences démontrées

- Data engineering et simulation de données.
- Traitement continu avec Pathway.
- Machine learning non supervisé avec scikit-learn.
- Industrialisation Python avec modules, CLI et tests.
- API REST avec FastAPI.
- Dashboard analytique avec Streamlit et Plotly.
- Persistance SQLite.
- Documentation technique et storytelling CV.
- Dockerisation d'une application Python.

## Limites honnêtes du projet

- Données entièrement simulées.
- Modèle non validé par un expert industriel.
- Pas de supervision métier réelle.
- Pipeline volontairement simple pour rester fiable en V1.
- Agent IA rule-based en première version, avec extension LLM prévue.
